# Roadmap

This roadmap is organized around evidence. Dates should be added after the Build Week deadline and team capacity are confirmed.

## Product Sprint

- Finalize mission, category, and language.
- Choose primary Builder and developer personas.
- Validate the value proposition with five to ten interviews.
- Design Builder journey and trust disclosures.
- Define prototype AI Credit rules.
- Choose the demo workload using verifiability and safety criteria.
- Produce low-fidelity UI mockups.

**Exit:** the team can state what is real, simulated, risky, and measurable.

## Architecture

- Freeze job, task, worker, receipt, and credit contracts.
- Define worker trust boundary and allow-list mechanism.
- Prototype Smart Contribution policies.
- Benchmark candidate workload on heterogeneous local workers.
- Write threat model and failure scenarios.
- Decide the minimum deployment stack.

**Exit:** one end-to-end spike proves task execution and verification.

## Build

- Implement coordinator and task state machine.
- Implement two workers and safe task runner.
- Add leasing, timeout, retry, and pause behavior.
- Implement verifier and redundant sampling.
- Implement append-only prototype credit ledger.
- Build Builder controls and live dashboard.
- Add test dataset, automated checks, and setup scripts.

**Exit:** acceptance criteria in [MVP.md](MVP.md) pass reliably.

## Demo

- Lock the 90-second story.
- Seed a deterministic public/synthetic job.
- Rehearse normal, paused-worker, and rejected-result paths.
- Capture proof of real multi-worker execution.
- Record primary video and fallback footage.
- Create architecture diagram and product screenshots.

**Exit:** a fresh machine or clean environment can run the demo from instructions.

## Devpost

- Align project story, technical claims, and implementation.
- Document how OpenAI tools were used, based on actual usage.
- Add repository, demo, images, team, and required links.
- Complete eligibility and submission checklist.
- Perform final accuracy, licensing, privacy, and brand review.

**Exit:** submission is complete, honest, and reproducible.

## After Build Week: Pilot

- Recruit 25–100 opt-in Builders.
- Test two workload classes.
- Measure total versus useful work and energy.
- Develop reputation and stronger verification.
- Test developer quickstart and pricing hypotheses.
- Explore partnerships for useful credit redemption.

## After Build Week: Network

- Harden cross-platform sandboxing.
- Add jurisdiction and data-class scheduling.
- Introduce auditable cooperative governance.
- Expand hardware and task compatibility only when safety evidence supports it.
- Publish network health, energy, reliability, and credit-economy reports.

## Decision gates

GridOne advances only when the previous stage demonstrates Builder trust, useful economics, verifiable work, and honest environmental accounting.
